# Interview Prep Tracks (15/30/60 Days)

These tracks are built for SRE, Production, Platform, and Systems interviews.
Pick one based on your timeline.

## How to Use

- Study one core topic daily
- Practice one debugging/system-design question daily
- Write concise verbal answers (2-4 minutes each)
- Do at least 2 mock interviews per week

## 15-Day Sprint (High Intensity)

Best when interviews are already scheduled.

### Week 1

1. Linux internals + networking basics
2. Kubernetes troubleshooting + deployment safety
3. SLI/SLO/error budget math
4. Distributed systems trade-offs
5. Incident management + postmortems
6. CI/CD reliability patterns
7. Full mock interview + feedback loop

### Week 2

1. Advanced observability (metrics/traces/logs)
2. Capacity planning and scaling strategy
3. Platform engineering and golden paths
4. Cloud reliability and multi-region design
5. Security fundamentals for SRE
6. Architecture interview dry run
7. Final revision + cheat sheets

## 30-Day Balanced Plan

Best for steady preparation with retention.

### Week 1: Foundations

- Linux process/memory/network fundamentals
- SQL + storage reliability basics
- SRE principles, toil, and automation

### Week 2: Cloud + Kubernetes

- K8s scheduling, probes, quotas, autoscaling
- Stateful workloads and failure scenarios
- Networking inside clusters (DNS, CNI, ingress)

### Week 3: System Design + Performance

- API and event-driven architecture design
- Caching, queueing, consistency, backpressure
- p95/p99 latency optimization playbook

### Week 4: Interview Simulation

- 3 end-to-end mocks (debugging + design + behavioral)
- postmortem-style feedback after each
- polish concise STAR stories

## 60-Day Mastery Track

Best for senior/staff-level interviews.

### Month 1

- complete all reliability foundations
- implement 2 mini projects:
  - synthetic monitoring dashboard
  - canary release gate with rollback logic

### Month 2

- advanced platform and architecture topics:
  - policy as code
  - multi-region failover
  - cost/reliability optimization
- 6+ high-quality mock interviews

## Daily Interview Routine (60-90 mins)

1. **20 min** revise concepts
2. **20 min** command-line/problem-solving drill
3. **20 min** system design prompt
4. **10-30 min** verbal answer recording and critique
