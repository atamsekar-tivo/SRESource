# Recommended Books to Crack Tech Interviews

This is a curated, high-signal list for SRE, systems, and software interviews.
Focus on depth over volume: it is better to master 8-10 books than skim 30.

## Tier 1 (Must Read)

### 1) Designing Data-Intensive Applications - Martin Kleppmann

- Best for distributed systems and system design interviews
- Read deeply: replication, partitioning, consistency, streaming
- Interview value: trade-offs and architecture clarity

### 2) Site Reliability Engineering - Google

- Core SRE foundations: SLOs, toil reduction, incident response
- Interview value: practical reliability thinking

### 3) The Site Reliability Workbook - Google

- Action-oriented follow-up to SRE book
- Interview value: implementation detail and operations maturity

### 4) Kubernetes Up & Running - Kelsey Hightower et al.

- Best practical K8s understanding for most interviews
- Interview value: platform and deployment confidence

### 5) System Design Interview (Vol 1 + Vol 2) - Alex Xu

- Great for structured design practice and communication
- Interview value: concise design articulation under time pressure

## Tier 2 (Strong Advantage)

### 6) Database Internals - Alex Petrov

- Deep understanding of storage engines and indexing
- Interview value: stronger backend and performance reasoning

### 7) Computer Networking: A Top-Down Approach - Kurose/Ross

- Strong networking fundamentals for debugging rounds
- Interview value: clear packet-to-application mental model

### 8) Linux Performance - Brendan Gregg

- Production performance debugging from first principles
- Interview value: elite troubleshooting answers

### 9) Release It! - Michael Nygard

- Stability patterns for real production systems
- Interview value: failure mode prevention and resilience design

### 10) Fundamentals of Software Architecture - Richards/Ford

- Architecture style and trade-off language
- Interview value: senior/staff-level design discussions

## Tier 3 (Leadership + Communication Edge)

### 11) The Manager's Path - Camille Fournier

- Helps with cross-team influence and leadership stories

### 12) Staff Engineer - Will Larson

- Excellent for senior/staff interview preparation

### 13) Thinking in Systems - Donella Meadows

- Improves systemic reasoning and feedback-loop analysis

## Suggested Reading Order

1. DDIA
2. SRE + SRE Workbook
3. Kubernetes Up & Running
4. System Design Interview Vol 1/2
5. Linux Performance + Networking
6. Database Internals + Release It!
7. Leadership books

## 8-Week Reading Strategy

### Weeks 1-2

- DDIA (core chapters)
- daily design prompt practice

### Weeks 3-4

- SRE + Workbook
- incident/reliability mock questions

### Weeks 5-6

- Kubernetes + Linux Performance
- debugging simulation drills

### Weeks 7-8

- System design books + leadership books
- full mock loops and answer polishing

## Final Advice

- Read actively and summarize in your own words.
- Convert each chapter into potential interview Q&A.
- Build a personal "decision log" of trade-offs you can reference in interviews.
